# Security Policy

## Supported versions

Mapture is still pre-`v1.0.0`.

The project currently supports:

- the latest stable release
- the rolling `canary` prerelease
- the current `main` branch for development only

Older prereleases may not receive fixes.

## Reporting a vulnerability

Please do not open a public issue for security problems.

Preferred path:

1. Use GitHub's private vulnerability reporting / security advisory flow if it is enabled for the repository.
2. If that is not available, contact the maintainers privately through GitHub.

Include:

- affected version or commit
- impact
- reproduction steps
- any proposed mitigation

## Response expectations

The project aims to:

- acknowledge reports quickly
- reproduce and assess the issue
- coordinate a fix before public disclosure when possible

Because Mapture is pre-1.0, fixes may ship as either a stable release or a canary build first, depending on severity and scope.
