# Code of Conduct

## Our standard

mapture.dev is an open-source project. We expect contributors and maintainers to keep the project respectful, constructive, and safe for people with different levels of experience, backgrounds, and viewpoints.

Examples of expected behavior:

- be respectful in technical disagreement
- assume good intent, but be direct about problems
- focus critique on code, behavior, and decisions
- help newcomers understand project context and constraints
- avoid harassment, insults, intimidation, and exclusionary behavior

Examples of unacceptable behavior:

- personal attacks
- trolling or deliberate disruption
- discriminatory or hateful language
- repeated bad-faith arguing after a decision is documented
- publishing private information without consent

## Scope

This code of conduct applies in:

- GitHub issues
- pull requests
- release discussions
- project documentation discussions
- other project-managed communication spaces

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers directly through GitHub. Use the support and repository contact channels in [SUPPORT.md](./SUPPORT.md). Security-related reports should follow [SECURITY.md](./SECURITY.md).

## Enforcement

Maintainers may remove comments, close discussions, reject contributions, or block participants when needed to protect the project and its community.

Enforcement decisions will prioritize safety, clarity, and the long-term health of the project.
