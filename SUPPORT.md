# Support

Use the right channel so the project stays easy to maintain.

## Bug reports

Open a GitHub issue with the bug report template and include:

- your command
- your fixture or repository structure
- the actual error output
- the expected behavior

## Feature requests

Open a GitHub issue with the feature request template.

Good requests explain:

- the problem you are trying to solve
- why the current workflow is not enough
- what outcome you need from Mapture

## Usage questions

If GitHub Discussions is enabled, use Discussions for general questions.

If Discussions is not enabled yet, open an issue and label it as a usage question.

## Security

For vulnerabilities or sensitive findings, follow [SECURITY.md](./SECURITY.md) instead of opening a public issue.
