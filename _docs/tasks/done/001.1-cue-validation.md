# Task 001.1: CUE Schema Validation Integration

## Goal
Implement data validation for `mapture.yaml` and the three catalog files (`teams.yaml`, `domains.yaml`, `events.yaml`) using the [CUE language](https://cuelang.org/) package (`cuelang.org/go`). This task replaces manual struct-based validation with robust, declarative CUE schemas.

## Context
Writing imperative validation code logic in Go to verify required fields, enum exact-matches, and regex strings (like kebab-cased IDs) is error-prone and tedious. By integrating CUE, we can declare our constraints elegantly alongside our Go types, letting `cuelang.org/go` handle the heavy lifting for Validation Layers 1 & 2 natively.

## Requirements

### 1. Install Dependencies
- Run `go get cuelang.org/go@latest` to add the CUE package to `go.mod`.

### 2. Define CUE Schemas (`internal/schema` or `src/internal/schema`)
- Create formal `.cue` files that mirror the types defined in `_docs/types/catalog-schemas.md` and `_docs/types/config-schema.md`.
- **Implement Enum Types:** E.g., `EventRole: "definition" | "trigger" | "listener" | "bridge-out" | "bridge-in" | "publisher" | "subscriber"`.
- **Implement Regex Constraints:** Enforce kebab-case rules (e.g., `id: string & =~"^[a-z0-9-]+$"`).
- **Implement Default Values & Optionals:** Ensure `scan.exclude`, for instance, defaults safely if omitted.

### 3. Build Go Validation Wrapper
- Set up a robust loader using `cuecontext.New()`.
- Unmarshal target YAML/JSON files, decode them into CUE values, and `Unify()` them against our loaded `.cue` schemas.
- If unification succeeds, decode the CUE values directly into our native Go structs.

### 4. User-Friendly Error Formatting
- When CUE detects schema violations (e.g., a missing required field or bad enum value), intercept the `cue.Error` and map it into the project's standard diagnostic warnings/errors formatted for the user.

## Definition of Done
- Native Go tests can parse a dynamically loaded `mapture.yaml` or `teams.yaml` snippet using CUE.
- Invalid test snippets (e.g., an event missing `role`, or providing `status: random`) fail compilation precisely at the CUE unification step with an easily understandable error.
- The `src/` repo is structured to efficiently embed or load `.cue` schemas along with the executable.
