# Task 001.0: Canonical Module Rename

## Goal
Update the Go module path to point to the canonical GitHub organization and repository (`github.com/mandotpro/mapture.dev`), ensuring all imports and binaries correctly resolve to the official OSS release path.

## Context
The project was originally scaffolded using a personal namespace (`github.com/angelmanchev/mapture` or similar). To successfully publish the CLI and provide a trustworthy open-source import path for any future Go API consumers, the module path must be aligned with the real GitHub project location before other development continues.

## Requirements

### 1. Update `go.mod`
- Change the first line of `go.mod` to: `module github.com/mandotpro/mapture.dev`.
- Verify Go version (`1.25.0` or as defined) is preserved.

### 2. Rename Internal Imports
- Perform a global find-and-replace across the entire `src/` directory (or wherever Go source files live) to update any absolute internal imports.
  - Old: `github.com/angelmanchev/mapture/...`
  - New: `github.com/mandotpro/mapture.dev/...`

### 3. Update Documentation
- Ensure any installation instructions (e.g. `go install github.com/mandotpro/mapture.dev@latest`) in `README.md` and other root documents reference this correct domain.

## Definition of Done
- `go build ./src/...` compiles correctly.
- `go test ./src/...` runs without import resolution errors.
- `go mod tidy` successfully cleans dependencies against the new module path without errors.
